# 方法论体系

> 从学习和实战中提炼出的、经过验证的方法体系
> 知识库是输入，方法论是输出。没有数据验证的方法不算方法论。

---

## 核心方法论

### [自我成长](./self-growth.md)

**目标设定、责任承担、持续成长**

- 目标设定的四个维度：梦想、价值观、目标、策略
- 责任 = 权力：把责任推给谁，就给了谁权力
- 竹子理论：扎根 4 年，90 天长到 20 米
- 好运的组成：储存资本、识别机会、果断行动
- 金钱观改造：改变信仰，创造奇迹

### [财务规划](./financial-planning.md)

**先保障，再安全，再自由。跳步是最大的风险。**

- 财务自由三阶段：保障→安全→自由
- 三阶段目标计算公式
- 50/50 还债原则
- 40:40:20 投资原则
- 预算管理和财务监控

### [收入增长](./income-growth.md)

**整天工作的人没有时间赚钱。目标是构建系统，不是出卖时间。**

- 收入层次模型：主动收入→半被动收入→被动收入
- 收入翻倍路径：主业+副业+投资
- 开源优先于节流
- 企业家收入路径

### [投资理财](./investment.md)

**投资是让钱为你工作，而不是你为钱工作。**

- 40:40:20 投资配置原则
- 投资组合配置指南
- 价值投资方法论
- 指数基金定投策略

### [产品策略](./product-strategy.md)

**软件产品是最接近"让钱工作"的路径之一。**

- MVP 原则：先验证核心需求，再扩展功能
- 变现路径优先级：订阅制>一次性买断>增值服务
- 产品定位和差异化

### [行动启动](./action-start.md)

**从"知道"到"做到"的最短路径。**

- 最小阻力启动法
- 不完美的开始，胜过完美的等待
- 从一次行动到持续行动

## 方法论 vs 知识库

| 维度 | 知识库 | 方法论 |
|------|--------|--------|
| 来源 | 外部输入（书籍、课程、研究） | 内部提炼（实践验证） |
| 状态 | 理论、原始 | 可操作、已验证 |
| 使用场景 | 学习、研究 | 执行、复用 |
| 更新频率 | 持续新增 | 验证后才写入 |

---

## 方法论更新规则

1. 只有经过实践验证的方法才能写入
2. 每条方法注明来源（学习资料 / 个人实践 / 数据验证）
3. 验证失败的方法标注原因，不删除，留作教训
4. 每季度复盘一次，淘汰过时方法，更新有效方法
